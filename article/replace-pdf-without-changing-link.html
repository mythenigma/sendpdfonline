Replace a PDF Without Changing the Link

Same URL, new file. You swap the PDF in the background; recipients keep using the original link and see the updated version. No re-sending.

Why This Matters

You shared a link; then you find an error. Normally you’d upload again, get a new link, resend — and the client asks which link to use. With MaiPDF: the link stays. You replace the file in the portal. Everyone who opens the same link now sees the new version.

MaiPDF only. Maiimg does not support file replacement.

How It Works

Same Link, New Content

The reading code (and URL) stays. You inject the new PDF and its settings into that code.

Use Cases

When to Use It

  • Fix mistakes — typo in a contract, wrong price. Replace and done.
  • Update versions — price list, catalog, training doc. Same link, new file.
  • Change settings — add email verification, view limit, or watermark via the new file.
  • Retract — replace with a blank PDF to effectively revoke content.

Tips

Quick Tips

Save your control codes (e.g. in a password manager). Test the new PDF in a separate link before replacing. MaiPDF only — Maiimg has no replace feature.

Execution

Implementation Checklist

Before You Share

  • Lock the final file version and remove draft pages.
  • Choose an access policy: open, verified, or restricted viewers.
  • Set a clear expiry window for time-sensitive documents.
  • Decide if download/print should be disabled for this audience.

After You Share

  • Check access records in the first 24 hours to confirm engagement.
  • Follow up when a key recipient has viewed the file multiple times.
  • Revoke or tighten access immediately if the link is forwarded.
  • Replace the file on the same link to avoid version drift.
